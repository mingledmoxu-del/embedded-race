# 选题风险分析表

## 1. 评估前提

本分析按以下前提进行：

- 参赛形式：`单人完成`
- 时间资源：课余推进，开发周期有限
- 常见能力模型：具备一定嵌入式开发基础，但不假设你同时精通硬件、算法、结构、工业总线、Linux 驱动和模型训练
- 目标：`优先拿奖，同时保证项目能做完、能稳定演示、能答辩讲清楚`

说明：

- 单人项目最怕的不是“题目不高级”，而是“系统链路太长，最后每个部分都只做到 60 分”。
- 对单人来说，最优策略通常不是追求最大题，而是追求`一个技术主线特别突出、其余部分能闭环`。

## 2. 风险评估维度

| 维度 | 含义 |
| --- | --- |
| 硬件门槛 | 是否需要复杂硬件、电机机构、定制板、多传感器协同 |
| 算法门槛 | 是否需要训练/部署视觉、语音、多模态模型 |
| 系统集成风险 | 是否涉及采集、推理、通信、控制、UI、回放等多链路耦合 |
| 调试周期风险 | 是否容易出现长时间排障、性能不稳定、现场翻车 |
| 展示效果 | 答辩时是否容易让评委直观看懂 |
| 单人可控性 | 一个人能否较稳地完成主线并打磨细节 |

风险等级说明：

- `低`：单人较容易控住
- `中`：可以做，但要收敛范围
- `高`：对单人明显吃力
- `很高`：除非你相关经验很强，否则不建议作为主选题

## 4. 一个人最不适合选的题目

以下题目不建议你作为单人主攻方向：

1. `ST 车规 MCU-智能飞行汽车`
原因：
- 需要飞行器、地面平台、控制、结构、安全防护、路径规划等多系统协同
- 硬件、结构和调试量远超单人舒适区

2. `NXP 视觉 + EtherCAT 的智能机器人`
原因：
- 视觉、工业总线、控制同步、稳定性验证全部叠在一起
- 单人很容易后期被集成问题拖死

3. `复杂机械臂 / 四足机器人 / 多电机高自由度机器人`
原因：
- 机械结构和控制联调成本很高
- 一旦结构件、电机、供电或驱动链路出问题，会严重拖慢整体进度

## 5. 一个人最适合的项目类型

单人更适合以下类型：

### 5.1 单场景视觉识别 + 联动控制

特点：

- 一个摄像头
- 一个主要识别目标
- 一个明确动作响应
- 一个可视化界面或告警系统

优点：

- 主线清晰
- 容易演示
- 算法和系统都能体现
- 不依赖复杂机械结构

代表题目：

- NXP 机械臂危险区域入侵检测
- NXP 仓储物流 AI 智能分拣系统
- RK 端侧 AI 视觉应用

### 5.2 轻量边缘 AI + 明确应用场景

特点：

- 不追求大而全
- 聚焦一个识别任务或异常检测任务
- 强调本地推理、低延迟、稳定运行

优点：

- 适合单人打磨细节
- 答辩时容易讲“为什么必须边缘侧处理”

代表题目：

- ST 嵌入式人工智能
- ESP 嵌入式边缘 AI 应用
- NXP AI / 机器学习方向

### 5.3 交互完整、功能克制的终端型项目

特点：

- 有屏幕
- 有 UI
- 有少量传感器
- 有联网或本地联动

优点：

- 容易做出完成度
- 观感通常比纯后台系统更好

代表题目：

- ESP 高性能图形人机交互系统
- ESP 智能交互驱动的 AIoT 应用系统
- NXP 智能家居 AI 监护与交互系统

## 6. 结合单人实际，推荐你优先考虑的项目

### 推荐 1：NXP 机械臂危险区域入侵检测

综合评价：`最适合单人冲奖`

为什么适合你一个人做：

- 可以不依赖真实机械臂，减少硬件负担
- 视觉识别目标明确，算法范围可控
- 系统结构容易收敛成完整闭环
- 工业安全价值强，评委容易理解
- 很容易做出“检测-告警-联动-回放”的答辩亮点

建议收敛方案：

- 只做单摄像头
- 只识别人手或人体其一
- 危险区先做静态区，再升级动态区
- 响应动作先做分级告警 + GPIO 模拟急停
- 增加事件截图、日志和回放页

### 推荐 2：NXP 仓储物流 AI 智能分拣系统

综合评价：`很适合单人`

为什么适合：

- 场景清晰，容易讲清楚产业价值
- 可从 OCR 或条码单主线切入，不必全做
- 不依赖复杂机构也能演示

风险：

- 如果场景数据不足，模型鲁棒性会受影响
- 要注意光照、反光、遮挡这类实际问题

### 推荐 3：ESP 高性能图形人机交互系统

综合评价：`最稳妥的单人完成型项目`

为什么适合：

- 开发门槛相对可控
- 容易做出完成度高的系统
- 如果你更擅长软件和 UI，而不是算法训练或机械结构，这个方向更稳

风险：

- 如果只是“普通中控屏”，冲奖力度不足
- 必须叠加一个鲜明应用场景，比如工业中控、智能仪表、康复训练终端

### 推荐 4：ST 智能可穿戴设备

综合评价：`适合愿意做硬件集成的单人`

为什么适合：

- 可做成紧凑型工程样机
- 容易体现低功耗、传感器融合、本地智能分析
- 比普通 IoT 项目更有产品感

风险：

- 如果你没有 PCB、结构和电源经验，推进会慢
- 演示效果通常不如视觉类项目直观

## 7. 结论

如果你是一个人参赛，我建议你优先按下面顺序考虑：

1. `NXP 机械臂危险区域入侵检测`
2. `NXP 仓储物流 AI 智能分拣系统`
3. `ESP 高性能图形人机交互系统`
4. `ST 智能可穿戴设备`

如果你想要的是：

- `最像一等奖项目`：选 `NXP 机械臂危险区域入侵检测`
- `最稳妥做完并做好展示`：选 `ESP 高性能图形人机交互系统`
- `更硬核、更有技术味`：选 `ST 智能可穿戴设备`

我当前最推荐你的单人方向仍然是：

`NXP 挑战创新选题二：机械臂危险区域入侵检测`

因为它在单人条件下，综合平衡了：

- 题目价值
- 系统完整度
- 演示效果
- 技术含量
- 可控风险
