# NXP机械臂危险区域入侵检测单人实施方案

## 1. 项目定位

项目名称建议：

`面向协作机器人安全防护的危险区域入侵检测与联动终端`

项目定位：

基于 NXP FRDM-IMX93 平台，构建一个用于工业协作场景的视觉安全防护终端。系统通过摄像头采集作业画面，检测人员或手部进入机械臂危险区域的行为，并触发分级告警、GPIO 模拟急停、日志记录和事件回放。

## 2. 单人开发目标

单人项目必须收敛，因此目标不能过大。

建议你把项目目标拆成三个层次：

### 2.1 基础目标

- 实时采集摄像头视频
- 在画面中配置危险区域
- 检测人体或手部进入危险区域
- 实现屏幕标注和声光告警

### 2.2 完整目标

- 进入危险区后触发 GPIO 模拟急停
- 生成事件截图和日志
- 支持历史事件回放
- 提供简单本地界面

### 2.3 冲奖目标

- 动态危险区
- 分级安全策略
- 遮挡恢复和抖动抑制
- 多规则管理
- 远程查看或 Web 端事件查看

## 3. 推荐系统架构

### 3.1 硬件组成

- 主控：FRDM-IMX93
- 摄像头：USB 摄像头或 MIPI 摄像头
- 显示：HDMI 显示器
- 告警模块：蜂鸣器、LED
- 联动输出：GPIO 模拟急停信号

### 3.2 软件模块

1. 视频采集模块
2. 图像预处理模块
3. 危险区域配置模块
4. 目标检测模块
5. 目标与区域关系判断模块
6. 告警与联动模块
7. 事件记录模块
8. 本地显示与设置模块

### 3.3 数据流

`摄像头输入 -> 目标检测 -> 危险区判断 -> 告警分级 -> GPIO/蜂鸣器/LED 联动 -> 事件截图与日志 -> 本地展示/回放`

## 4. 功能收敛建议

## 4.1 第一阶段只做人或手其中一种

不要一开始就同时做：

- 人体检测
- 手部检测
- 姿态估计
- 多目标跟踪

单人建议优先顺序：

1. 人体检测
2. 手部检测
3. 再考虑细粒度动作判断

理由：

- 人体检测更稳
- 更容易先把闭环做出来

## 4.2 危险区建议先做静态多边形

第一版建议：

- 在画面上框一个矩形或多边形危险区
- 只判断目标框中心点或底部关键点是否进入

第二版再升级：

- 多区域
- 不同风险等级
- 动态区域

## 4.3 联动先做模拟急停

不要一开始就接真实机械臂控制器。

先做：

- LED 红灯
- 蜂鸣器
- GPIO 输出高低电平
- 屏幕上弹出急停状态

这样能把安全联动逻辑做完整，又避免真实控制风险。

## 5. 推荐技术路线

### 5.1 视觉检测

建议选轻量目标检测模型，重点是稳定，不是追求最大模型。

优先方向：

- 人体检测轻量模型
- 手部检测轻量模型

建议原则：

- 优先找已有轻量模型
- 优先部署成熟推理链路
- 不要把大部分时间花在重新训练复杂模型上

### 5.2 判定逻辑

核心逻辑建议：

- 识别目标框
- 计算目标位置点
- 判断是否进入危险区
- 做时间窗滤波，避免抖动误报

建议加入：

- 连续多帧确认后再触发
- 离开危险区后延时解除

### 5.3 事件管理

每次告警记录：

- 时间
- 目标类别
- 告警等级
- 截图路径
- 是否触发急停

如果精力允许，可加：

- 短视频片段
- 事件回放页面

## 6. 答辩亮点设计

单人项目想拿高分，必须有几个明确亮点。

建议重点打下面 5 个点：

1. `工业安全场景真实`
2. `边缘端实时检测，低延迟响应`
3. `危险区域可配置`
4. `分级告警和联动急停`
5. `事件记录与可追溯分析`

如果还能再加一个点，优先加：

`遮挡和抖动条件下的误报抑制`

## 7. 演示方案建议

答辩演示不要太复杂，建议流程固定：

1. 展示正常监控状态
2. 人员在安全区外活动，系统不报警
3. 人员或手进入危险区
4. 画面标框、区域高亮、蜂鸣器告警、LED 亮起
5. GPIO 模拟急停触发
6. 自动生成事件截图和日志
7. 打开历史记录页面查看事件

这个流程很顺，评委一看就懂。

## 8. 单人开发中最容易踩的坑

1. 一开始就追求多模型、多目标、多规则
2. 一开始就接复杂机械结构
3. 过早优化 UI，主链路没跑通
4. 过度依赖训练新模型
5. 没有做误报抑制和日志系统

## 9. 最小可行版本

如果你时间紧，最少要保证下面这些内容做扎实：

- 实时视频输入
- 单一目标检测
- 单危险区域入侵判断
- 声光告警
- GPIO 模拟急停
- 事件截图与日志

只要这几项稳定，已经是一个完整项目。

## 10. 最终建议

这个项目的正确做法不是把它做成“复杂算法比赛”，而是把它做成一个：

`稳定、完整、可演示、可追溯的工业安全终端`

单人参赛时，稳定性和完整度通常比花哨功能更重要。
